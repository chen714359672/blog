关于npm包的发布流程

# 前言

  1. 因为公司后台项目筛选条件都是由后端php通过window.page_data写入进来，统一封装成组件来调用了。

  2. 但后面因为项目越来越多，每次筛选条件组件都得重新从其他项目复制过来引入，很麻烦，

  3. 因此决定把组件做成npm包，直接上传到npm上，后面的项目就可以直接install再全局引入了

# 发布-步骤一

那肯定是先在npm官网注册一个账号了，没有账号也不可能发布npm包

npm的官网是https://www.npmjs.com/

# 发布-步骤二

创建vue组件，本地创建一个项目文件，进入文件中

然后执行

npm init
1

按照提示操作就行了，但这时候的项目命名不要很随便

最好先在npm官网上也搜索下是否存在相同的名称，因为如果存在同名的话，是发布不成功的。

个人建议的话按照公司名称-组件名来起名是最能区分的

# 发布-步骤三

生成的目录结构的话如下：（其中的template是自定义的筛选样式）

需要重点注意的就是index.js跟package.json这两个文件了

index.js的话需要是注册组件并抛出组件

import FiltersTpl from '../src/filters-tpl.vue'

const TableWrapper = {
    FiltersTpl
}

//注册组件
Object.keys(TableWrapper).forEach(name => {
    TableWrapper[name].install = function(Vue) {
      Vue.component(name, TableWrapper[name]);
    };
    // Vue.component(name, TableWrapper[name])
})
  
export default TableWrapper
export {
    FiltersTpl
}

# 发布-步骤四

弄完上面的，就万事俱备只欠东风了。直接在文件的根目录中，输入

npm login
1

登录刚才在npm官网注册的账号密码，然后发布

npm publish --access=public
1

发布成功后，就可以在npm官网上找到自己发布的npm包了

# 更新

更新了组件的内容，需要手动修改package.json中的version版本号。按照惯例就是小改动是1.0.0->1.0.1啦，大的更新就是1.0.0->2.0.0，然后再重新发布一边即可
